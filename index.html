🐳 Docker Lab — Cómo Iniciar

Contenedores ligeros. Reinicia en segundos. El laboratorio perfecto.

Paso 1: Instalar Docker

  • macOS: brew install --cask docker
  • Linux: sudo apt install docker.io
  • Windows: Docker Desktop
  • Verificar: docker --version

Paso 2: Descargar Labs

  • DVWA: vulnerables/web-dvwa
  • Juice Shop: bkimminich/juice-shop
  • WebGoat: webgoat/webgoat
# Iniciar DVWA
$ docker run -d -p 8080:80 vulnerables/web-dvwa
→ http://localhost:8080 (admin/password)
# Iniciar Juice Shop
$ docker run -d -p 3000:3000 bkimminich/juice-shop
→ http://localhost:3000
[DOOM TIP] Docker te permite resetear el lab en segundos: docker rm -f y vuelve a correr.
[DOOM] Docker = rapidez. No pierdas tiempo instalando, empieza a hackear.

🕷️ DVWA — Damn Vulnerable Web App

El laboratorio clásico de Doom. SQLi, XSS, LFI y más.

Vulnerabilidades incluidas

  • SQL Injection — clásico y blind
  • XSS — reflected, stored, DOM
  • LFI/RFI — inclusión de archivos
  • CSRF — Cross-Site Request Forgery
  • Command Injection — OS commands
# Iniciar y configurar
$ docker run -d -p 8080:80 vulnerables/web-dvwa
$ firefox http://localhost:8080
→ Login: admin / password
→ Setup → Create Database
→ Security Level: Low → Medium → High

4 niveles de dificultad

[DOOM TIP] Empieza en Low, sube a Medium, luego High. Impossible es para graduarte.
[DOOM] DVWA en "impossible" = el verdadero reto. Medium es para rendirse.

📦 VirtualBox — Metasploitable

Máquinas virtuales completas. El campo de batalla clásico de Doom.

Paso a paso

  • 1. Descargar VirtualBox
  • 2. Descargar Metasploitable 2 (.vmdk)
  • 3. New VM → Linux → Ubuntu 64-bit
  • 4. Use existing disk → seleccionar .vmdk
  • 5. Network: Host-Only Adapter
# Encontrar IP de Metasploitable
$ nmap -sn 192.168.56.0/24
→ 192.168.56.101 (Metasploitable)
# Escanear servicios
$ nmap -sV 192.168.56.101
→ 21/ftp, 22/ssh, 80/http, 3306/mysql...

Login: msfadmin/msfadmin

[DOOM TIP] Usa Host-Only Adapter para aislar la VM. NUNCA la pongas en Bridged.

🔴 VulnHub — VMs Descargables

Cientos de máquinas vulnerables creadas por la comunidad. Infinitas.

Cómo empezar

  • 1. Ir a vulnhub.com
  • 2. Filtrar por dificultad: Easy → Medium
  • 3. Descargar .ova o .vmdk
  • 4. Importar en VirtualBox o VMware
  • 5. Encontrar IP con nmap -sn

VMs recomendadas para empezar

  • Kioptrix Level 1 — ideal para principiantes
  • Mr-Robot — CTF temático, divertido
  • DC-1 — Drupal exploitation
  • Stapler — múltiples vectores
  • Bulldog — Django vulnerable
